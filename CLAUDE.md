# StockAnalyse · 股票见底打分评估系统

> 本文件是项目的权威规格说明（source of truth）。需求、规则、打分细则、数据源、参数全部以此为准。
> 最后更新：2026-05-31

---

## 1. 项目目标

输入一个**股票代码或名称**，依据 `StockAnalyseStandard.docx` 的规则：
1. **自动采集**该股票所需的全部行情与指标数据；
2. **逐项打分**，汇总出满分 100 的「见底（抄底时机）评分」+ 仓位建议。

### 运行模式（两种）
- **模式 A · 单股即时查询**：给一个代码 → 立即产出打分报告。
  例：`python score.py 600519`
- **模式 B · 批量 + 定时任务**：给一份股票列表（watchlist），设为每天固定时段自动跑，把结果
  **① 发邮件到 Gmail（你配置的收件邮箱）** 且 **② 同时在本地生成 md 报告**。

### 市场范围
**A股（沪深）+ 港股**（暂定这两个市场）。

---

## 2. 评分框架总览（来自 docx）

满分 **100**，分 4 个维度，每维度 **25** 分。总分对应仓位建议：

| 总分 | 结论 | 仓位建议 |
|---|---|---|
| ≥80 | 极佳，见底概率极高 | 重仓 50–80% |
| 70–79 | 良好，调整充分 | 分批买入 30–50% |
| 60–69 | 及格，调整接近尾声 | 轻仓试探 10–20% |
| 50–59 | 偏弱，继续观望 | 等待更多信号 |
| <50 | 差，调整未充分 | 不动 |

## 3. 打分细则 v1.0（已与用户确认）

> docx 只列了"评分项"未给分值，以下分值经用户确认。`*` 标注的字段为**仅采集、不参与评分**的前置信息。

### 维度一 · 时间维度（25）— 调整时间是否充分
**仅采集不评分**：`*前期高点日期`、`*当前日期`、`*调整天数`（= 当前日期 − 前期高点日期，按交易日或自然日见 §6）

| 子项 | 满分 | 评分规则 |
|---|---|---|
| 斐波那契时间窗命中 | **10** | 调整天数命中斐波那契数（±2天容差），按命中的数给分（保持比例）：<br>89/55→10 ｜ 34→9.3 ｜ 21→8.0 ｜ 13→6.7 ｜ 8→5.3 ｜ 5→4.0 ｜ 超出容差按到最近 Fib 点的距离线性折减 |
| 黄金比例时间关系 | **8** | 调整天数 ÷ 前期上涨天数 ≈ 0.618 或 1.618：±0.05→8 ｜ ±0.1→4.6 |
| 15天小周期重合 | **7** | 调整天数 ≈ 15 的整数倍（±1天）→ 7（正式打分项，非加分） |

### 维度二 · 空间维度（25）— 价格是否回撤到关键支撑
**仅采集不评分**：`*前期上涨起点(元)`、`*前期上涨高点(元)`、`*上涨幅度(元)`

| 子项 | 满分 | 评分规则 |
|---|---|---|
| 黄金分割回撤位 | **7** | 当前价贴近某黄金回撤位（±1.5%）：0.5 / 0.618（黄金坑）→7 ｜ 0.382→4.7 ｜ 0.786→3.9 |
| 重要均线支撑 | **7** | 当前价贴近且不破关键均线，长期均线权重高，命中累加封顶 7：<br>250日→2.3 ｜ 120日→1.9 ｜ 55日→1.6 ｜ 20日→0.8 ｜ 5日→0.4（周线 5/20/55 另计，并入累加） |
| 支撑价位接近 | **7** | 当前价贴近三档支撑（±1.5%）之一，每档 ≈2.3：第一支撑=前期低点 ｜ 第二=筹码密集区 ｜ 第三=整数关口/趋势线 |
| 三重支撑重合 | **4** | 黄金位 + 均线 + 支撑价位 三者价格高度重合（都在 ±2% 内）→ 4（正式打分项，非加分） |

### 维度三 · 量价维度（25）— 底部缩量、低换手、底部形态
**仅采集不评分**：`*当前成交量(万股)`、`*5日均量(万股)`

| 子项 | 满分 | 评分规则 |
|---|---|---|
| 缩量比例 <50% | **6** | 当前量 < 5日均量×50% → 6 ｜ 50–70% → 3 |
| 换手率达标 | **6** | <1%(大盘) / <3%(小盘) → 6 ｜ 略高(阈值1.5倍内) → 2.6。**港股若无换手率数据则弱化/缺省**（见 §6） |
| 底部K线形态 | **8** | 强信号（放量阳线 / 阳包阴 / 底部堆量）各 4；蓄势信号（地量十字星 / 缩量横盘）各 2；命中累加封顶 8 |
| 量价配合 | **5** | 上涨放量(+2.5) + 下跌缩量(+2.5) → 封顶 5（正式打分项，非加分） |

### 维度四 · 技术指标（25）— 多周期超卖共振
> **⚠️ 偏离 docx：docx 写的是 RSI(14)，经用户确认改为 RSI(12)。** 其余指标参数见 §5。

| 子项 | 满分 | 评分规则 |
|---|---|---|
| RSI(12) 日 / 60分 / 15分 | **9** | 每周期 3 分：数值 <30 给 1.5 + 拐头向上给 1.5 |
| MACD 日 / 60分 / 15分 | **9** | 每周期 3 分：绿柱缩短 +1 + 金叉 +1 + 底背离 +1 |
| KDJ | **4** | J 值 <0 后拐头向上 → 4 |
| 布林带 | **3** | 跌破下轨后重新站回 → 3 |

**核对**：维度一 10+8+7=25 ｜ 维度二 7+7+7+4=25 ｜ 维度三 6+6+8+5=25 ｜ 维度四 9+9+4+3=25 ｜ **合计 100** ✓

## 4. 数据源方案（组合方案，已确认）

没有单一免费库能同时覆盖 A股+港股 的分钟线+换手率，故采用**组合 + 兜底**：

| 数据 | 首选 | 兜底 | 说明 |
|---|---|---|---|
| A股 日线/周线（含换手率、MA250深度） | **baostock** | akshare(sina) | baostock 免注册，`turn` 字段直接给换手率，日线轻松拉 2 年+ |
| A股 60分钟 / 15分钟 | **baostock** | akshare(sina) | 实测 15分钟可达 2.4 年 9280 根，深度最佳 |
| 港股 日线（MA250） | **yfinance** | akshare(东财) | 实测 2 年深度，满足 MA250 |
| 港股 60分钟 | **yfinance** | — | 实测 2 年深度 |
| 港股 15分钟 | **yfinance**（最近60天） | — | 仅算"当前"15分钟指标，60天足够（不需长历史） |
| 港股 换手率 | akshare(东财港股日线) | 自算/弱化 | yfinance 无换手率字段；港股侧该因子可弱化 |
| 技术指标 RSI/MACD/KDJ/布林 | **pandas-ta 自算** | 自实现 | 所有库都不内置；避开 TA-Lib（Windows 编译坑） |

### 关键注意事项
- **复权**：技术分析一律用**前复权**（baostock `adjustflag=2`，akshare `adjust="qfq"`，yfinance `auto_adjust`）。否则除权跳空污染 MA250 与前高前低识别。
- **换手率口径不一致**：baostock `turn` = 百分比（0.6118 即 0.61%）；akshare sina `turnover` = 小数（0.006118）。混用必须统一为百分比。
- **本机网络**：当前 Windows 环境有本地代理（127.0.0.1:7897）会掐断**东方财富历史K线接口**。A股走 baostock/sina 可绕开；若用 akshare 东财接口需在代码里禁用系统代理（`session.trust_env=False`，清空 proxies）。
- **限频/重试**：yfinance 偶发 429、东财接口易限频，数据层需加**重试 + 退避**。
- **依赖**：`pip install baostock yfinance akshare pandas pandas-ta`

## 5. 技术指标计算口径

| 指标 | 参数 | 备注 |
|---|---|---|
| 均线 MA | 5/20/55/120/250 日；5/20/55 周 | 简单移动平均，前复权价 |
| RSI | **周期 12**（非14） | 日 / 60分 / 15分 三周期 |
| MACD | 12 / 26 / 9（标准） | 日 / 60分 / 15分；"绿柱缩短"=DIF-DEA 柱体连续收窄；"金叉"=DIF 上穿 DEA；"底背离"=价格创新低但 MACD 未创新低 |
| KDJ | 9,3,3（标准） | 关注 J<0 后拐头向上 |
| 布林带 | 20, 2σ | 跌破下轨后收盘价重新站回轨内 |

## 6. 关键参数（已确认）

| 参数 | 取值 | 说明 |
|---|---|---|
| 前期高点查找窗口 | **半年（≈120 交易日）** | 在该窗口内找最高点为"前期高点"，再从该高点往前找最近显著低点为"上涨起点"（峰谷检测） |
| 大盘/小盘阈值 | **流通市值 300 亿** | ≥300亿=大盘（换手率阈值 1%）；<300亿=小盘（阈值 3%） |
| 价格贴近容差 | **±1.5%** | 黄金位、均线、支撑位"贴近"判定 |
| 三重支撑重合容差 | **±2%** | 三者价位都落在 ±2% 区间内 |
| 斐波那契天数容差 | **±2 天** | |
| 黄金比例时间容差 | ±0.05（满分）/ ±0.1（半分） | |
| 15天小周期容差 | ±1 天 | |
| 调整天数口径 | 交易日（待实现时确认；自然日为备选） | |

## 7. 输出与交付

- **本地 md 报告**：每次打分生成一份 markdown，存到 `reports/`（命名含代码+日期）。包含：采集到的原始数据、逐项得分明细、四维度小计、总分、仓位建议。
- **Gmail 邮件**：模式 B 批量跑完后，把当批所有股票的报告汇总发到 你配置的收件邮箱。
  - 发送用 `smtplib` + Gmail SMTP，**需要 Gmail「应用专用密码」(App Password)**，存在本地配置（不进版本库）。
- **定时任务**：Windows 任务计划程序（`schtasks`）每天固定时段调用批量入口。

## 8. 计划架构（实现时遵循）

```
StockAnalyse/
├── CLAUDE.md                 # 本规格文件
├── StockAnalyseStandard.docx # 原始规则文档
├── requirements.txt
├── config.example.yaml       # 配置模板（阈值/邮件/参数）；真实 config.yaml 不进库
├── watchlist.txt             # 模式B的股票列表
├── src/
│   ├── datasource/           # 统一数据接口；ashare(baostock+akshare)、hk(yfinance+akshare)
│   ├── indicators.py         # RSI(12)/MACD/KDJ/布林/均线（pandas-ta + 自实现）
│   ├── features.py           # 前期高低点、调整天数、黄金分割、缩量、换手率、K线形态识别
│   ├── scoring/              # time/space/volume/tech 四维度 + aggregate 汇总
│   ├── report.py             # 生成 md 报告
│   └── notifier.py           # Gmail 发送
├── score.py                  # 模式A入口：python score.py <code>
├── run_batch.py              # 模式B入口：批量 + 定时
└── reports/                  # 本地报告输出
```

**分层**：数据层（取数、标准化、前复权）→ 指标层（算指标）→ 特征层（识别前高/缩量/形态等）→ 打分层（四维度规则）→ 报告/通知层。

## 9. 实现状态与用法

**已完成（v1，真实数据实测跑通 A股+港股）**：
- 数据层 `src/datasource/`：baostock(A股) + yfinance(港股)，前复权、换手率、流通市值反算
- 指标层 `src/indicators.py`：RSI(12)/MACD/KDJ/布林/均线
- 特征层 `src/features.py`：峰谷检测、黄金分割、缩量、K线形态、背离等（阈值集中在 `PARAMS`）
- 打分层 `src/scoring/`：四维度 + 汇总 + 仓位建议（细则 v1.0）
- 报告层 `src/report.py` + 模式A入口 `score.py`
- 模式B `run_batch.py` + 通知 `src/notifier.py` + 定时 `setup_schedule.ps1`

**用法**：
```bash
python score.py 600519          # 模式A：单股查询，出本地 md
python score.py 0700.HK
python run_batch.py             # 模式B：批量打分 watchlist.txt，出汇总 md +（配好后）发邮件
# 定时：powershell -ExecutionPolicy Bypass -File .\setup_schedule.ps1 -Time 18:00
```
邮件：复制 `config.example.yaml` 为 `config.yaml`，填 Gmail **应用专用密码**、`enabled: true`。

## 10. 待调优 / 未决项

1. **峰谷检测显著性阈值**（最重要）：当前用"半年内最高 + 往前最低"的简单 argmax/argmin。
   短上涨腿等边界情形（如 600519 出现 7 天上涨腿、time_ratio=10）会失真，需加显著性过滤。
2. **K线形态阈值**（放量倍数 1.5、地量 0.6、十字星实体 0.25 等，见 `features.PARAMS`）待用真实样本校准。
3. **底背离**回溯窗口（默认 60 根，前后半段比较）较粗，可改用真实摆动低点。
4. **调整天数**目前用交易日（非自然日）。
5. **港股换手率**：当前统一弱化为缺省 3 分；如需精确，可接 akshare 东财港股日线补数。
6. **Gmail App Password**：需用户提供后才能联调真实邮件发送。
